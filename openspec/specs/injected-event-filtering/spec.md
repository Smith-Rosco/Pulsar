# injected-event-filtering Specification

## Purpose
Ensures the low-level keyboard hook can distinguish between injected (synthetic) keyboard events from `SendInput`/`keybd_event` and physical keystrokes, enabling precise modifier state tracking during synthetic event suppression windows.

## Requirements
### Requirement: GlobalKeyboardHook SHALL distinguish injected from physical keyboard events

The low-level keyboard hook callback SHALL read the `flags` field from `KBDLLHOOKSTRUCT` (offset 8 in `lParam`) and extract the `LLKHF_INJECTED` bit (0x10) to determine whether the current event was generated by `SendInput`, `keybd_event`, or another software injection mechanism.

#### Scenario: Injected key down event is identified
- **WHEN** a keyboard event arrives via `SendInput` or `keybd_event`
- **THEN** the `LLKHF_INJECTED` flag in `KBDLLHOOKSTRUCT.flags` SHALL be set (bit 4 = 1), and the hook SHALL classify the event as injected

#### Scenario: Physical key down event is identified
- **WHEN** a keyboard event arrives from a physical key press
- **THEN** the `LLKHF_INJECTED` flag SHALL be cleared (bit 4 = 0), and the hook SHALL classify the event as physical

#### Scenario: Injected key up event is identified
- **WHEN** a keyboard release event arrives via `SendInput` or `keybd_event`
- **THEN** the `LLKHF_INJECTED` flag SHALL be set, and the hook SHALL classify the event as injected

### Requirement: UpdateModifierTracker SHALL skip only injected events during synthetic suppression

When `_syntheticEventSuppression` is active (between `OnSyntheticEventBegin` and `OnSyntheticEventEnd`), `UpdateModifierTracker` SHALL skip modifier state updates ONLY for injected events. Physical keyboard events (both KeyDown and KeyUp) SHALL continue to update the internal modifier tracker regardless of suppression state.

#### Scenario: Physical Shift KeyUp updates tracker during suppression
- **WHEN** `_syntheticEventSuppression` is true AND a physical Shift KeyUp event arrives
- **THEN** `UpdateModifierTracker` SHALL set `_trackedShiftDown` to false

#### Scenario: Injected Alt KeyDown is skipped during suppression
- **WHEN** `_syntheticEventSuppression` is true AND an injected Alt KeyDown event arrives
- **THEN** `UpdateModifierTracker` SHALL NOT update `_trackedAltDown` (early return)

#### Scenario: Physical events always update tracker regardless of suppression
- **WHEN** `_syntheticEventSuppression` is false
- **THEN** `UpdateModifierTracker` SHALL process ALL events (both injected and physical) normally

### Requirement: UpdateModifierTracker SHALL run before event listener pre-filter

The hook callback's modifier tracker update SHALL execute before the listener pre-filter check, ensuring modifier state is maintained even when no event listeners are subscribed.

#### Scenario: Tracker updates even without KeyUp listeners
- **WHEN** a modifier KeyUp event arrives AND `OnKeyUp` has no subscribers
- **THEN** `UpdateModifierTracker` SHALL still update the tracker state before the event is passed through without listener notification

#### Scenario: Tracker order in hook callback
- **WHEN** any keyboard event is processed by `HookCallback` with `nCode >= 0`
- **THEN** `UpdateModifierTracker` SHALL be called BEFORE the pre-filter check for event listener nullity
