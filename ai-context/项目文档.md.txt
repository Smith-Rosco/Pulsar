#Pulsar 项目技术文档 (Project Pulsar)> **当前状态**: Phase 1-5 (核心架构重构 & 基础 UI 实现) - [已完成]
> **技术栈**: C# / .NET 8 / WPF / MVVM / Dependency Injection
> **架构模式**: MVVM (Model-View-ViewModel) + Service-Oriented
> **目标**: 在保留 GridCaster 核心功能的基础上，通过解耦架构提升代码的可维护性、可测试性和扩展性。

##1. 🏗️ 项目全景与蓝图 (Project Blueprint)* **项目名称**: Pulsar (脉冲星)
* **项目愿景**: 构建一个基于现代 WPF 技术栈的系统级效率轮盘。通过严格的 MVVM 架构实现 UI 与逻辑分离，利用依赖注入管理服务生命周期，提供极致的响应速度与“果冻般”的视觉体验。

###核心架构变革 (Architecture Evolution)| 特性 | 旧版 GridCaster | **新版 Pulsar** |
| --- | --- | --- |
| **架构模式** | Monolithic (UI 逻辑耦合在 Code-behind) | **Strict MVVM** (UI 与逻辑完全分离) |
| **依赖管理** | 静态类 / 单例 (Static/Singleton) | **依赖注入容器** (Microsoft.Extensions.DI) |
| **数据流** | 直接操作 UI 控件 | **数据绑定 (Data Binding) + INotifyPropertyChanged** |
| **动画实现** | C# 代码控制 Storyboard | **XAML DataTrigger + 样式动画** (数据驱动) |
| **模型设计** | 混合了 View 逻辑的 Model | **POCO Model** (纯数据) + **ViewModel** (状态) |

###技术栈细节* **Runtime**: .NET 8 (Windows)
* **GUI Framework**: WPF (主框架) + WinForms (仅用于钩子/SendKeys 兼容)
* **DI Container**: `Microsoft.Extensions.DependencyInjection`
* **Config Parsing**: `Tomlyn` (TOML 配置文件支持)
* **Input Hook**: P/Invoke User32.dll (封装为 `GlobalKeyboardHook` 服务)

##3. 📜 技术契约与规范 (Technical Contracts)###3.1 命名空间与冲突解决 (Critical)由于开启了 `<UseWindowsForms>true</UseWindowsForms>`，导致 WPF 与 WinForms 存在大量同名类型冲突。
**解决方案**: 必须在 `GlobalUsings.cs` 中强制指定类型映射，严禁在业务代码中混用。

```csharp
// GlobalUsings.cs 示例
global using WpfApplication = System.Windows.Application;
global using GridItem = Pulsar.Models.GridItem; // 强制指向我们的 Model
global using UserControl = System.Windows.Controls.UserControl; // 强制指向 WPF 控件

```

###3.2 动画驱动机制 (Animation Strategy)* **禁止**: 在 ViewModel 或 Code-behind 中创建 `Storyboard` 对象。
* **推荐**: 在 XAML 的 `<Style.Triggers>` 中监听 ViewModel 的 `bool IsActive` 属性。
* **优势**: ViewModel 不需要引用任何 UI 库，便于单元测试；UI 线程负担更小。

###3.3 异步与并发 (Async/Await)* 所有 I/O 操作（配置文件读写、外部进程启动）必须使用 `async/await`。
* `GlobalKeyboardHook` 事件处理中应避免耗时操作，必要时使用 `Task.Run` 或 `Dispatcher` 封送。

##4. 🧩 核心数据模型 (Core Models)Pulsar 将数据 (Model) 与 状态 (ViewModel) 进行了严格分离。

**Model (纯数据, 用于序列化):**

```csharp
public class GridItem
{
    public GridItemType Type { get; set; }
    public int Slot { get; set; }
    public string Label { get; set; }
    public string Cmd { get; set; }
    public string? Process { get; set; }
}

```

**ViewModel (视图状态, 用于绑定):**

```csharp
public class SlotViewModel : ViewModelBase
{
    // 静态属性 (构造时计算)
    public double CenterX { get; }
    public double CenterY { get; }
    
    // 动态属性 (触发通知)
    private bool _isActive;
    public bool IsActive 
    { 
        get => _isActive; 
        set => SetProperty(ref _isActive, value); // 触发 View 的动画 Trigger
    }
}

```

##5. 🛠️ 故障排查指南 (Troubleshooting)* **Q: 运行后看不到界面，只有后台进程？**
* **A**: 这是设计预期。Pulsar 是托盘程序，启动后默认隐藏。请尝试按下热键 **`Ctrl + Q`** 唤醒轮盘。


* **Q: 唤醒后能看到文字但看不到球体？**
* **A**: 检查 `JellyOrb.xaml.cs` 构造函数中是否调用了 `InitializeComponent()`。如果缺失，控件内容将不会被加载。


* **Q: 编译报错 "ambiguous reference between ..."?**
* **A**: 检查 `GlobalUsings.cs` 是否包含了冲突类型的强制映射（如 `GridItem`, `GridItemType`, `UserControl`）。


* **Q: 修改了 XAML 里的动画没有生效？**
* **A**: 确保你修改的是 `<Style.Triggers>` 里的 `DataTrigger`，并且 ViewModel 的 `IsActive` 属性确实发生了变化（可以通过断点调试 ViewModel）。
