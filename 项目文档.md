## 3. 📜 技术契约与规范 (Technical Contracts)

### 3.1 独立组件架构 (Independent Component Architecture)

* **去耦合**: 基础 UI 组件（如 `JellyOrb`）**严禁**依赖隐式的 `DataContext` 继承。
* **显式驱动**: 必须使用 `DependencyProperty` 定义输入接口（如 `IconKey`），并在 XAML 中通过 `RelativeSource` 或 `ElementName` 绑定自身属性。
* **自包含逻辑**: 图标解析（String -> ImageSource/Glyph）逻辑下沉至控件内部 (`OnIconKeyChanged`)，ViewModel 仅负责传递原始字符串 Key。

### 3.2 命名空间管理 (Namespace Management)

* **消除二义性**: 当同时引用 `System.Windows` (WPF) 和 `System.Windows.Forms` (WinForms) 时，必须使用**全限定名**（如 `Microsoft.Win32.OpenFileDialog`, `System.Windows.Controls.Button`）来避免编译器混淆 (`CS0104`)。
* **资源顺序**: 在 XAML 中，`StaticResource` (如 Converter) 必须在被引用之前定义（通常放在 `<UserControl.Resources>` 且位于最顶部）。

### 3.3 数据响应性 (Data Reactivity)

* **实时编辑**: 任何需要在 UI 上（如 SettingsWindow）实时修改并反馈的模型类（如 `GridItemBase`），**必须**继承 `ObservableObject` 并使用 `SetProperty`，否则选择图标后界面不会刷新。

## 5. 🛠️ 故障排查备忘 (Troubleshooting Memory)

* **Q: 编译报错 `CS0104` "Button/Application 是不明确的引用"?**
* **A**: 同时引入了 WPF 和 WinForms 命名空间。**Fix**: 删除不必要的 `using`，或在代码中使用全名 `System.Windows.Controls.Button`。


* **Q: XAML 设计器报错 `XDG0008` "命名空间中不存在 JellyOrb"?**
* **A**: 这是 C# 编译失败导致的副作用。**Fix**: 优先修复所有 C# 错误 (如 CS0104)，然后执行 "Clean Solution" -> "Build Solution"，XAML 错误会自动消失。


* **Q: 运行时报错 "无法找到名为 BooleanToVisibilityConverter 的资源"?**
* **A**: 资源定义在 XAML 树中的位置太靠后。**Fix**: 将 `<UserControl.Resources>` 移到 XAML 根节点下的第一行。


* **Q: 图标选择器选完后，主界面或设置页图标不刷新?**
* **A**: 数据模型 (`GridItemBase`) 是普通的 POCO 类，没有实现通知。**Fix**: 继承 `ObservableObject`，将属性 setter 改为 `SetProperty`。
