## 1. 🏗️ 项目全景与蓝图 (Project Blueprint)

* **项目名称**: Pulsar (脉冲星)
* **项目代号**: Pulsar Redux
* **核心价值**: 消除“切换”、“执行”与“输入”之间的微秒级阻力。
* **版本目标**: 构建一个逻辑严格分离、执行绝对稳定、交互符合直觉的系统级效率轮盘。

### 核心架构变革 (Architecture Evolution)

| 特性 | 旧版 Pulsar (v1.x) | **Pulsar Redux (v2.0)** | **Pulsar PKI (v3.0)** |
| --- | --- | --- | --- |
| **交互逻辑** | 单一混合逻辑 | **双模态物理隔离** (Launcher vs Smart Command) | **上下文感知注入** (Context Aware Injection) |
| **架构模式** | 单体服务 | 分层架构 (Layered) | **模块化单体 (Modular Monolith)** (Core + Features) |
| **窗口管理** | 标准 Show/Hide | **驻留幽灵模式** (Resident Ghost + `WS_EX_TOOLWINDOW`) | 保持不变 |
| **焦点管理** | 简单的 Activate | **条件性归还机制** (Conditional Focus Return) | **焦点回旋镖 (Focus Boomerang)** (Return -> Wait -> Inject) |
| **执行逻辑** | 仅启动或仅切换 | **幂等执行**: Switch ➔ Fail ➔ Launch | **混合注入序列**: ActionRegistry 分发 -> PkiHandler |
| **配置体验** | 手动输入进程名 | **所见即所得** (WYSIWYG Process Picker) | **凭据保险箱** (DPAPI Encrypted Vault) |

## 3. 📜 技术契约与规范 (Technical Contracts)

### 3.1 模块化单体策略 (Modular Monolith Strategy)

* **注册中心模式**: 废弃硬编码的 `switch-case`，使用 `ActionRegistry` 进行依赖倒置。
* **Features 隔离**: 业务逻辑（如 PKI）必须封装在 `Features/` 目录，通过 `App.xaml.cs` 注册到 DI 容器。
* **无反射调用**: 显式注册 `registry.Register<TItem>(handler)`，确保编译期类型安全，避免运行时反射性能损耗。

### 3.2 线程安全与 UI 操作 (Thread Safety)

* **后台执行原则**: `CommandService` 默认使用 `Task.Run` 在后台线程执行 Handler，防止阻塞 UI 动画。
* **跨线程 UI 操作**:
* Handler (如 `PkiHandler`) **禁止**直接操作 UI 元素。
* **必须**通过 `IWindowService.RegisterHideAction` 暴露委托。
* **关键**: `RadialMenuWindow` 在实现该委托时，**必须**使用 `this.Dispatcher.Invoke(() => Dismiss())` 将操作调度回 UI 线程，否则会触发 `System.InvalidOperationException`。



### 3.3 驻留窗口与焦点管理 (Resident Window & Focus)

* **隐身注入**: 必须在 `OnSourceInitialized` 通过 `SetWindowLong` 注入 `WS_EX_TOOLWINDOW`。
* **上下文感知 (Context Awareness)**: `WindowService` 充当全局状态机，负责在 `Summon` 时记录 `PreviousWindowHandle`。
* **焦点回旋镖 (Focus Boomerang) - PKI 专用**:
1. `WindowService.HideMainWindow()` (触发 UI 线程隐藏)
2. `SetForegroundWindow(Previous_Handle)` (归还焦点)
3. `await Task.Delay(100)` (**必须**: 等待 Windows 动画与输入队列就绪)
4. `SendKeys` (注入文本)



### 3.4 数据安全 (Data Security)

* **加密存储**: `SecretItem` 中的密码字段必须存储为 `EncryptedData` (Base64)。
* **DPAPI**: 使用 `System.Security.Cryptography.ProtectedData` (Scope: CurrentUser)。
* **禁止明文**: 配置文件中不得出现明文密码。

## 5. 🛠️ 故障排查备忘 (Troubleshooting Memory)

* **Q: 触发 PKI 功能时程序崩溃，报错 `InvalidOperationException`？**
* **Cause**: `PkiHandler` 运行在后台线程，试图调用 `Dismiss()` 修改 `Opacity` 等 UI 属性。
* **Fix**: 在 `RadialMenuWindow` 注册 `RegisterHideAction` 时，包裹 `this.Dispatcher.Invoke(...)`。


* **Q: 注入时文字乱码或丢失？**
* **Cause**: 焦点归还后的 `Task.Delay` 时间过短，输入焦点还没完全回到目标窗口。
* **Fix**: 增加等待时间至 100ms~150ms。


* **Q: 窗口明明 Opacity=0，但在 Alt+Tab 里还是能看到一个透明的框？**
* **Fix**: 必须通过 P/Invoke 注入 `WS_EX_TOOLWINDOW` 样式。


* **Q: 成功切换或启动软件后，窗口“闪回”？**
* **Fix**: 使用 `ActionExecuted` 标志位控制焦点归还逻辑。
