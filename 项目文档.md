## 3. 📜 技术契约与规范 (Technical Contracts)

### 3.1 独立组件架构 (Independent Component Architecture)

* **去耦合**: 基础 UI 组件（如 `JellyOrb`）**严禁**依赖隐式的 `DataContext` 继承。
* **显式驱动**: 必须使用 `DependencyProperty` 定义输入接口（如 `IconKey`），并在 XAML 中通过 `RelativeSource` 或 `ElementName` 绑定自身属性。
* **自包含逻辑**: 图标解析（String -> ImageSource/Glyph）逻辑下沉至控件内部 (`OnIconKeyChanged`)，ViewModel 仅负责传递原始字符串 Key。

### 3.2 高性能动画 (High-Performance Animation)

* **帧级驱动**: 对于高频、连续的交互动画（如鼠标跟随），**严禁**使用 `DispatcherTimer` 或 `Storyboard`（会导致 GC 压力和卡顿）。必须使用 `CompositionTarget.Rendering` 事件，直接在 GPU 渲染前的每一帧更新 UI 属性。
* **插值算法**: 为避免物理引擎带来的震荡（Jitter），优先使用 **Lerp (线性插值)** 算法 (`Current + (Target - Current) * Factor`) 实现平滑跟随。

### 3.3 全局坐标系 (Global Coordinate System)

* **全屏响应**: 为了实现无死角的鼠标响应，必须抛弃 WPF 窗口内的相对坐标，转而使用**屏幕绝对像素坐标** (`System.Windows.Forms.Cursor.Position` vs `Visual.PointToScreen`) 进行计算，仅在最后一步赋值给 `TranslateTransform` 时除以 DPI Scale 转换回 WPF 逻辑单位。
* **命名空间二义性**: 当同时引用 `System.Windows` (WPF) 和 `System.Windows.Forms` (WinForms) 时，必须在文件头部明确 `using Point = System.Windows.Point;`。

## 5. 🛠️ 故障排查备忘 (Troubleshooting Memory)

* **Q: 动画出现高频抖动或“乱跳”?**
* **A**: 使用物理模拟（弹簧/阻尼）时，如果阻尼不足或帧率不稳定，会导致过冲震荡。**Fix**: 改用无惯性的 **Lerp (线性插值)** 算法，平滑因子设为 `0.1` 左右，实现温和的延迟跟随。
* **Q: 鼠标移出窗口后，控件反向位移?**
* **A**: 坐标计算依赖了窗口相对坐标，鼠标负值导致计算错误。**Fix**: 改用 `Forms.Cursor.Position` (屏幕绝对坐标) 减去 `Control.PointToScreen` (控件绝对坐标) 计算向量，彻底消除坐标系误差。
* **Q: 编译报错 `CS0104` "Point 是不明确的引用"?**
* **A**: 同时引入了 WPF 和 WinForms/Drawing 命名空间。**Fix**: 在文件头部添加 `using Point = System.Windows.Point;`。
* **Q: XAML 设计器报错 `XDG0008` "命名空间中不存在 JellyOrb"?**
* **A**: 这是 C# 编译失败导致的副作用。**Fix**: 优先修复所有 C# 错误 (如 CS0104)，然后执行 "Clean Solution" -> "Build Solution"，XAML 错误会自动消失。
