## 1. 🏗️ 项目全景与蓝图 (Project Blueprint)

* **项目名称**: Pulsar (脉冲星)
* **项目代号**: Pulsar Redux
* **核心价值**: 消除“切换”与“执行”之间的微秒级阻力。
* **版本目标**: 构建一个逻辑严格分离、执行绝对稳定、交互符合直觉的系统级效率轮盘。

### 核心架构变革 (Architecture Evolution)

| 特性 | 旧版 Pulsar (v1.x) | **Pulsar Redux (v2.0)** |
| --- | --- | --- |
| **交互逻辑** | 单一混合逻辑 | **双模态物理隔离** (Launcher vs Smart Command) |
| **判定算法** | 简单的碰撞检测 | **无限扇区极坐标算法** (Infinite Sector Selection) |
| **窗口管理** | 标准 Show/Hide | **驻留幽灵模式** (Resident Ghost + `WS_EX_TOOLWINDOW` Injection) |
| **焦点管理** | 简单的 Activate | **条件性归还机制** (Conditional Focus Return) 防止操作闪回 |
| **执行逻辑** | 仅启动或仅切换 | **幂等执行 (Idempotency)**: 自动判定 Switch ➔ Fail ➔ Launch |
| **配置体验** | 手动输入进程名 | **所见即所得** (WYSIWYG Process Picker w/ Icon Persistence) |

## 3. 📜 技术契约与规范 (Technical Contracts)

### 3.1 独立组件架构 (Independent Component Architecture)

* **去耦合**: 基础 UI 组件（如 `JellyOrb`）**严禁**依赖隐式的 `DataContext` 继承。
* **显式驱动**: 必须使用 `DependencyProperty` 定义输入接口。

### 3.2 跨线程与原生资源 (Interop & Thread Safety)

* **后台 UI 对象冻结**: 在非 UI 线程创建的 `BitmapSource` 必须调用 `.Freeze()`。
* **原生图标提取**: 使用 `SHGetFileInfo` 后必须立即 `DestroyIcon`。

### 3.3 驻留窗口模式 (Resident Window Pattern)

* **隐身注入**: 必须在 `OnSourceInitialized` 生命周期通过 `SetWindowLong` 注入 `WS_EX_TOOLWINDOW` 扩展样式。
* *目的*: 欺骗 DWM，使窗口虽然 `Visible` 但不出现在 Alt+Tab 列表和任务栏中。
* **焦点归还 (Focus Restoration)**:
* **Borrow**: `Summon()` 时通过 `GetForegroundWindow` 记录当前的活跃窗口句柄。
* **Conditional Return**: `Dismiss()` 时，**必须检查 `ActionExecuted` 标志位**。
* 若 `ActionExecuted == true` (已执行命令)：**禁止**归还焦点（让新窗口自然浮起）。
* 若 `ActionExecuted == false` (用户取消)：**必须**显式调用 `SetForegroundWindow` 将焦点还给之前的窗口。


* *目的*: 维护操作系统的 Z-Order 堆栈，同时防止“闪回”Bug。

### 3.4 数据绑定规范 (Data Binding Standard)

* **Model 响应式**: 所有绑定到 UI 的 Model 类（如 `LauncherItem`）必须继承自 `ObservableObject` 并使用 `SetProperty`。
* *原因*: 防止后台逻辑更新数据（如选择进程后回填）时，UI 界面无感知。

### 3.5 图标处理策略 (Icon Handling Strategy)

* **提取与缓存**: 从运行中进程提取的图标必须持久化为 PNG 文件存储在 `AppData/Pulsar/Cache/Icons`，禁止仅在内存中传递。
* **智能加载 (Smart Load)**:
* **图片文件 (.png, .jpg)**: 必须直接读取像素数据加载 (`BitmapImage`)。
* **可执行文件 (.exe, .lnk)**: 必须调用 `ExtractAssociatedIcon` 提取资源。
* *原因*: 避免使用 `ExtractAssociatedIcon` 读取 PNG 时，错误地返回了系统默认图片查看器（如 WPS、Photos）的图标。

## 5. 🛠️ 故障排查备忘 (Troubleshooting Memory)

* **Q: 窗口明明 Opacity=0，但在 Alt+Tab 里还是能看到一个透明的框？**
* **A**: 仅设置 `ShowInTaskbar=False` 对顶级窗口无效。**Fix**: 必须通过 P/Invoke 注入 `WS_EX_TOOLWINDOW` 样式。
* **Q: 关闭轮盘后，按下 Alt+Tab 切换到了当前窗口（而不是上一个窗口）？**
* **A**: Pulsar 抢占了焦点但未归还。**Fix**: 在 `Dismiss` 时调用 `SetForegroundWindow(previousHandle)`。
* **Q: 成功切换或启动软件后，窗口“闪”了一下又回到了原来的窗口（闪回）？**
* **A**: `Dismiss` 逻辑过于尽责，在执行命令后依然归还了焦点。**Fix**: ViewModel 引入 `ActionExecuted` 标志，View 层 `Dismiss` 方法检查此标志，仅在未执行动作时归还焦点。
* **Q: 打开选择器弹窗时崩溃，提示 IOException?**
* **A**: 引用了不存在的默认图片。**Fix**: 使用 Unicode 字符或 XAML 绘图作为 Fallback。
* **Q: 选择进程后，Process Name 文本框没有变化，但数据实际已更新？**
* **A**: `LauncherItem` 是普通的类，没有实现 `INotifyPropertyChanged`。**Fix**: 继承 `ObservableObject`。
