## 3. 📜 技术契约与规范 (Technical Contracts)

### 3.1 独立组件架构 (Independent Component Architecture)

* **去耦合**: 基础 UI 组件（如 `JellyOrb`）**严禁**依赖隐式的 `DataContext` 继承。
* **显式驱动**: 必须使用 `DependencyProperty` 定义输入接口（如 `IconKey`）。
* **动态资源**: 所有颜色画笔必须使用 `DynamicResource` 引用，禁止硬编码 Hex 值，以支持运行时主题切换。

### 3.2 资源管理与主题 (Resource Management)

* **纯净字典**: `ResourceDictionary` 文件（如 `Theme.Dark.xaml`）**严禁**包含 `x:Class` 属性或 `.xaml.cs` 代码隐藏文件。它们必须是纯 XAML 资源，否则会导致 `CS0111` 类型定义冲突。
* **窗口级隔离**: 主题切换应针对特定 `Window` 实例操作（修改 `window.Resources.MergedDictionaries`），而非全局 `App.Current.Resources`，以支持不同窗口使用不同主题（如深色轮盘 + 浅色设置）。
* **光效控制**: 对于不适合浅色模式的特效（如 Blur/Glow），应在资源字典中定义控制变量（如 `Theme.Orb.BlurOpacity`），通过透明度或数值绑定来“软关闭”特效，避免在代码中写 `if (LightMode)`。

### 3.3 依赖注入原则 (DI Principles)

* **服务优先**: 严禁在构造函数中注入纯数据模型（如 `AppConfig`）。View 和 ViewModel 必须注入提供数据的服务（如 `IConfigService`），并在初始化阶段异步获取数据。
* *错误*: `public Window(AppConfig config)` -> 导致崩溃。
* *正确*: `public Window(IConfigService service)` -> `service.LoadAsync()`。



### 3.4 全局坐标系 (Global Coordinate System)

* **全屏响应**: 必须使用屏幕绝对像素坐标 (`System.Windows.Forms.Cursor.Position`) 进行计算。
* **渲染循环**: 任何涉及鼠标跟随或高频交互的窗口，在重构时必须确保 `CompositionTarget.Rendering` 循环被正确恢复，否则会导致交互失效。

## 5. 🛠️ 故障排查备忘 (Troubleshooting Memory)

* **Q: 编译报错 `CS0111` "Type 'Theme' already defined"?**
* **A**: 资源字典文件 (`Theme.x.xaml`) 意外包含了 `.xaml.cs` 代码文件或 `x:Class` 属性。**Fix**: 删除 `.cs` 文件并移除 XAML 头部的 `x:Class` 定义。


* **Q: 启动崩溃 `Unable to resolve service for type 'AppConfig'`?**
* **A**: 构造函数请求了未注册的数据模型。**Fix**: 改为注入 `IConfigService`，并异步加载配置。


* **Q: 轮盘唤起后无法交互，且位置偏移?**
* **A**: 重构代码时意外删除了 `UpdateLoop` (Rendering) 和 `UpdateDpiAndPosition` 逻辑。**Fix**: 恢复渲染循环和坐标计算逻辑，确保 ViewModel 能接收到鼠标数据。


* **Q: 鼠标移出窗口后，控件反向位移?**
* **A**: 坐标计算依赖了窗口相对坐标。**Fix**: 改用 `Forms.Cursor.Position` (屏幕绝对坐标) 计算向量。


* **Q: 编译报错 `CS0104` "Point 是不明确的引用"?**
* **A**: 冲突的命名空间。**Fix**: 添加 `using Point = System.Windows.Point;`。
