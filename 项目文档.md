## 3. 📜 技术契约与规范 (Technical Contracts)

### 3.1 独立组件架构 (Independent Component Architecture)

* **去耦合**: 基础 UI 组件（如 `JellyOrb`）**严禁**依赖隐式的 `DataContext` 继承。
* **显式驱动**: 必须使用 `DependencyProperty` 定义输入接口（如 `IconKey`）。
* **动态资源**: 所有颜色画笔必须使用 `DynamicResource` 引用，以支持运行时主题切换。

### 3.2 资源管理与主题 (Resource Management)

* **纯净字典**: `ResourceDictionary` 文件必须是纯 XAML，严禁包含 `x:Class` 或 Code-behind，避免 `CS0111` 错误。
* **窗口级隔离**: 主题切换应修改特定 `Window.Resources.MergedDictionaries`，实现不同窗口不同主题。
* **光效控制**: 通过透明度资源（如 `Theme.Orb.BlurOpacity`）来“软关闭”特效，避免在代码中进行硬逻辑判断。

### 3.3 数据一致性与服务 (Data Consistency & Services)

* **事件驱动更新**: `IConfigService` 必须暴露 `ConfigUpdated` 事件。所有依赖配置数据的 ViewModel (如 `RadialMenuViewModel`) 必须订阅此事件并重新执行 `LoadAsync`，严禁仅在构造函数中读取一次数据。
* **服务优先**: View/ViewModel 必须注入 Service 接口，严禁直接注入数据 Model。

### 3.4 进程命名规范 (Process Naming Convention)

* **无后缀原则**: `WindowService` 返回的 `ProcessName` 不包含 `.exe` (如 "msedge", "notepad")。
* **键值纯净**: 在 `SettingsViewModel` 保存 Profile 时，**必须移除** `.exe` 后缀，确保存储的 Key 与 `ProcessName` 精确匹配。
* **小写标准化**: 所有进程名比较和存储必须转换为小写 (`ToLower()`)。

### 3.5 全局坐标系 (Global Coordinate System)

* **全屏响应**: 必须使用 `System.Windows.Forms.Cursor.Position` 获取绝对坐标。
* **循环恢复**: 重构时必须检查 `CompositionTarget.Rendering` 循环是否存活。

## 5. 🛠️ 故障排查备忘 (Troubleshooting Memory)

* **Q: 设置修改后，轮盘内不生效（数据不同步）?**
* **A**: ViewModel 是 Singleton，仅在构造时读取了旧数据。**Fix**: 在 `IConfigService` 增加 `ConfigUpdated` 事件，ViewModel 订阅该事件并在触发时重新加载数据。
* **Q: 添加 "edge" 配置后，轮盘在 Edge 浏览器中无法识别?**
* **A**: `Process.ProcessName` 返回的是无后缀的 `msedge`，而配置存成了 `edge.exe`。**Fix**: 1. 用户需使用正确的进程名 (msedge)。2. 代码在保存时自动移除 `.exe` 后缀，确保 Key 格式一致。
* **Q: 编译报错 `CS0111` "Type 'Theme' already defined"?**
* **A**: 资源字典包含 Code-behind。**Fix**: 删除 `.xaml.cs` 并移除 XAML 中的 `x:Class`。
* **Q: 启动崩溃 `Unable to resolve service for type 'AppConfig'`?**
* **A**: 构造函数注入了 Model。**Fix**: 改为注入 `IConfigService`。
* **Q: 轮盘唤起后无法交互/位置偏移?**
* **A**: 丢失渲染循环。**Fix**: 恢复 `CompositionTarget.Rendering` 和 DPI 计算逻辑。
