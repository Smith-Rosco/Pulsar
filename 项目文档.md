## 1. 🏗️ 项目全景与蓝图 (Project Blueprint)

* **项目名称**: Pulsar (脉冲星)
* **项目代号**: Pulsar Redux
* **核心价值**: 消除“切换”、“执行”与“输入”之间的微秒级阻力。
* **版本目标**: 构建一个逻辑严格分离、执行绝对稳定、交互符合直觉的系统级效率轮盘。

### 核心架构变革 (Architecture Evolution)

| 特性 | 旧版 Pulsar (v1.x) | **Pulsar Redux (v2.0)** | **Pulsar PKI (v3.0)** |
| --- | --- | --- | --- |
| **交互逻辑** | 单一混合逻辑 | **双模态物理隔离** (Launcher vs Smart Command) | **上下文感知注入** (Context Aware Injection) |
| **架构模式** | 单体服务 | 分层架构 (Layered) | **模块化单体 (Modular Monolith)** (Core + Features) |
| **窗口管理** | 标准 Show/Hide | **驻留幽灵模式** (Resident Ghost + `WS_EX_TOOLWINDOW`) | 保持不变 |
| **焦点管理** | 简单的 Activate | **条件性归还机制** (Conditional Focus Return) | **焦点回旋镖 (Focus Boomerang)** (Capture -> Return -> Wait -> Inject) |
| **执行逻辑** | 仅启动或仅切换 | **幂等执行**: Switch ➔ Fail ➔ Launch | **混合注入序列**: ActionRegistry 分发 -> PkiHandler |
| **配置体验** | 手动输入进程名 | **所见即所得** (WYSIWYG Process Picker) | **凭据保险箱** (Split Storage + Encrypted Vault) |

## 3. 📜 技术契约与规范 (Technical Contracts)

### 3.1 模块化单体与多态策略

* **注册中心模式**: 使用 `ActionRegistry` 进行依赖倒置，解耦 Item 类型与执行逻辑。
* **多态序列化**: 在 `GridItemBase` 使用 `[JsonDerivedType]` 显式注册所有子类（Launcher/Command/Secret），防止 `System.NotSupportedException`。
* **ViewModel 绑定**: 基类 `GridItemBase` 必须包含用于 UI 状态的通用属性（如 `IsRecommended`），即使该属性不被序列化 (`[JsonIgnore]`)，以避免 WPF `BindingExpression` 错误。

### 3.2 存储与并发策略 (Storage Strategy)

* **分离存储 (Split Storage)**:
* `appsettings.json`: 仅存储 UI 布局结构 (`SecretItem` 的 ID、Label、Icon)。
* `secrets.json`: 存储敏感数据 (`SecretPayload` 的 Account、EncryptedData)，通过 ID 关联。


* **数据注水 (Hydration)**: `RadialMenuViewModel` 加载时，需并行读取两个文件，并在内存中将 Payload 填充回 Item。
* **并发写入 (Race Condition)**: 在保存时，**必须**先保存 `secrets.json` 并释放文件锁，**再**保存 `appsettings.json`（触发重载事件）。若并行保存 (`Task.WhenAll`) 且重载事件立即触发读取，会导致 `IOException`。

### 3.3 驻留窗口与焦点管理

* **上下文感知 (Context Awareness)**: `RadialMenuViewModel.Show()` 被触发的瞬间（显示前），**必须**调用 `WindowService.SetPreviousWindow` 捕获当前的前台窗口句柄。
* **焦点回旋镖 (Focus Boomerang)**:
1. `PkiHandler` 开始执行。
2. `HideMainWindow()` (Dispatcher Invoke)。
3. `GetPreviousWindow()` 获取之前捕获的句柄。
4. `SetForegroundWindow` 强制归还焦点。
5. `await Task.Delay(100)` (缓冲时间)。
6. `SendKeys`。



### 3.4 数据安全 (Data Security)

* **DPAPI**: 使用 `CurrentUser` 作用域。
* **故障排查**: 若 `Decrypt` 返回空字符串，通常意味着数据文件被复制到了不同机器/用户下，需重新录入。

## 5. 🛠️ 故障排查备忘 (Troubleshooting Memory)

* **Q: 保存配置时报错 `System.IO.IOException: The process cannot access the file ... secrets.json`？**
* **Cause**: `SettingsViewModel` 并行执行保存 (`Task.WhenAll`)，主配置保存完立即触发了重载事件，导致读取操作与尚未完成的写入操作冲突。
* **Fix**: 改为顺序执行：`await SaveSecrets(); await SaveConfig();`，并在 `SecretRepository` 增加重试机制。


* **Q: 保存时报错 `System.NotSupportedException: Runtime type 'SecretItem' is not supported...`？**
* **Cause**: `System.Text.Json` 默认不支持多态序列化。
* **Fix**: 在基类 `GridItemBase` 上添加 `[JsonDerivedType(typeof(SecretItem), "secret")]`。


* **Q: 控制台大量输出 `BindingExpression path error: 'IsRecommended' property not found on 'CommandItem'`？**
* **Cause**: `SettingsWindow` 复用了 `JellyOrb` 控件，但绑定的 Item（如 `CommandItem`）缺少该属性。
* **Fix**: 将 `IsRecommended` 属性提升至基类 `GridItemBase`，并标记 `[JsonIgnore]`。


* **Q: PKI 触发后程序无反应，焦点没有切回编辑器？**
* **Cause**: 轮盘呼出时未捕获上下文，导致 `PkiHandler` 拿到的 `PreviousWindowHandle` 为 `IntPtr.Zero`。
* **Fix**: 在 `RadialMenuViewModel.Show()` 方法首行调用 `WindowService.SetPreviousWindow(GetForegroundWindow())`。


* **Q: Global Command 下的 PKI 报错 `EncryptedData is EMPTY` 或解密失败？**
* **Cause**: 数据注水（Hydration）失败，通常是 ID 不匹配或配置残留。
* **Fix**: 在设置中删除该 Item 并重新添加（生成新 ID）。
