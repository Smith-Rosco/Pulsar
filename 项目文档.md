## 3. 📜 技术契约与规范 (Technical Contracts)

### 3.1 独立组件架构 (Independent Component Architecture)

* **去耦合**: 基础 UI 组件（如 `JellyOrb`）**严禁**依赖隐式的 `DataContext` 继承。
* **显式驱动**: 必须使用 `DependencyProperty` 定义输入接口（如 `IconKey`）。
* **动态资源**: 所有颜色画笔必须使用 `DynamicResource` 引用，以支持运行时主题切换。

### 3.2 驻留窗口模式 (Resident Window Pattern) [New]

* **生命周期**: 主窗口 (`RadialMenuWindow`) 单例启动后 **永不销毁 (`Close`) 或 隐藏 (`Hide`)**。仅通过 `Opacity` (0/1) 和 `IsHitTestVisible` (False/True) 控制显隐与交互。
* **预热机制**: `App.OnStartup` 必须强制调用一次 `Show()`，以确保窗口句柄 (HWND) 和 GPU 资源在用户首次交互前已分配完毕。
* **时序控制**: `Summon` (唤起) 的顺序必须是：更新位置 -> 启用交互 -> **强制 `UpdateLayout()**` -> 设置 `Opacity = 1`。
* **数据消毒**: `Dismiss` (关闭) 时必须调用 `ViewModel.ClearVisuals()` 清空 UI 内容，防止下一次渲染帧出现旧数据的“视觉残影”。

### 3.3 资源管理与主题 (Resource Management)

* **纯净字典**: `ResourceDictionary` 文件必须是纯 XAML，严禁包含 `x:Class` 或 Code-behind，避免 `CS0111` 错误。
* **窗口级隔离**: 主题切换应修改特定 `Window.Resources.MergedDictionaries`，实现不同窗口不同主题。

### 3.4 数据一致性与服务 (Data Consistency & Services)

* **事件驱动更新**: `IConfigService` 必须暴露 `ConfigUpdated` 事件。所有依赖配置数据的 ViewModel (如 `RadialMenuViewModel`) 必须订阅此事件并重新执行 `LoadAsync`。
* **服务优先**: View/ViewModel 必须注入 Service 接口，严禁直接注入数据 Model。

### 3.5 进程命名规范 (Process Naming Convention)

* **无后缀原则**: `WindowService` 返回的 `ProcessName` 不包含 `.exe` (如 "msedge", "notepad")。
* **键值纯净**: 在 `SettingsViewModel` 保存 Profile 时，**必须移除** `.exe` 后缀，确保存储的 Key 与 `ProcessName` 精确匹配。
* **小写标准化**: 所有进程名比较和存储必须转换为小写 (`ToLower()`)。

### 3.6 全局坐标系 (Global Coordinate System) [Upd]

* **全屏响应**: 必须使用 `System.Windows.Forms.Cursor.Position` 获取绝对坐标。
* **无限判定**: 只要鼠标距离中心超过 `DeadZone`，无论多远（即使在屏幕边缘）都必须计算角度并响应扇区激活，严禁设置最大距离限制。

## 5. 🛠️ 故障排查备忘 (Troubleshooting Memory)

* **Q: 轮盘唤起时有短暂的旧图标残影（Ghosting）?**
* **A**: 显存缓冲区未刷新。**Fix**: 1. 关闭时调用 `ClearVisuals()` 清空内容。 2. 开启时先 `UpdateLayout()` 强制重绘一帧空内容或新内容，然后再将 `Opacity` 设为 1。
* **Q: 首次按下快捷键轮盘显示慢?**
* **A**: WPF 窗口首次 `Show()` 需要初始化 DirectX 上下文。**Fix**: 在 `App.OnStartup` 中提前 `Show()` 窗口（Opacity=0），让其在后台驻留。
* **Q: 鼠标移出轮盘一段距离后无法触发动作?**
* **A**: 判定逻辑中存在 `dist < maxDist` 限制。**Fix**: 移除该限制，实现无限扇区。
* **Q: 设置修改后，轮盘内不生效（数据不同步）?**
* **A**: ViewModel 是 Singleton，仅在构造时读取了旧数据。**Fix**: 订阅 `ConfigUpdated` 事件重新加载。
* **Q: 添加 "edge" 配置后，轮盘在 Edge 浏览器中无法识别?**
* **A**: `Process.ProcessName` 返回的是无后缀的 `msedge`，而配置存成了 `edge.exe`。**Fix**: 保存时自动移除 `.exe` 后缀。
