# 产品需求文档 (PRD): Pulsar Key Injector (PKI)

**版本**: v1.0
**状态**: 待开发
**优先级**: P0 (High)
**适用架构**: Modular Monolith (基于 Pulsar v2.9+)

## 1. 项目背景与目标

**Pulsar** 是一个基于 C# WPF 的高频生产力工具。当前计划开发一个名为 **PKI (Pulsar Key Injector)** 的内部模块。
**目标**: 解决用户在 RDP (远程桌面) 和遗留工业软件 (如 OSF) 中无法保存密码、需频繁手动输入的痛点。
**核心机制**: 利用 Pulsar 现有的“驻留窗口”特性，实现 **呼出 -> 选择凭据 -> 归还焦点 -> 模拟键盘注入** 的自动化流程。

---

## 2. 核心功能需求 (Functional Requirements)

### 2.1 凭据管理 (Credential Management)

* **存储**: 必须支持存储多个凭据（Alias, Username, Password）。
* **安全**: 敏感字段（Password）在磁盘上必须是**加密**状态，内存中使用 `SecureString` 或在短暂解密后立即销毁。
* **配置入口**: 用户需要在设置界面通过简单的表格录入凭据。

### 2.2 注入执行 (The Injection Sequence)

* **触发**: 用户在 Pulsar 轮盘中点击某个代表凭据的 Item。
* **焦点回旋镖 (Focus Boomerang)**: 这是最关键的逻辑。
1. Pulsar 隐藏自身。
2. Pulsar 强制将系统焦点归还给 **上一个活跃窗口** (Target Window)。
3. Pulsar 等待目标窗口完成切换动画 (Wait Buffer)。
4. Pulsar 发送按键序列。


* **输入序列**: 默认采用 `[Clear] -> [Type User] -> [Tab] -> [Type Password] -> [Enter]` 的标准序列。

## 3. 技术实现细节 (Technical Implementation)

### 3.1 模块化架构 (Directory & Namespace)

必须在现有的 Pulsar 解决方案中遵循 **Modular Monolith** 模式：

* **路径**: `Pulsar/Features/Pki/`
* **新增类文件**:
* `Pki/Models/SecretItem.cs`: 数据模型。
* `Pki/Services/CredentialManager.cs`: 负责加密/解密/读写 JSON。
* `Pki/Services/InjectorService.cs`: 负责执行注入逻辑。


* **核心层变更**:
* `Core/Interfaces/ILauncherAction.cs`: 定义通用接口 (见下文)。



### 3.2 接口定义 (Interfaces)

为了解耦 `CommandService`，需定义执行动作的抽象接口：

```csharp
// File: Core/Interfaces/ILauncherAction.cs
public interface ILauncherAction
{
    // 判定该动作处理器是否支持某种指令类型 (例如 "PKI", "EXE")
    bool CanHandle(string actionType);
    
    // 执行具体逻辑
    // item: 包含配置信息的对象
    Task ExecuteAsync(LauncherItem item);
}

```

### 3.3 数据模型与安全 (Data & Security)

**SecretItem.cs**:

```csharp
public class SecretItem : ObservableObject
{
    public string Id { get; set; } = Guid.NewGuid().ToString();
    public string Alias { get; set; }      // 显示名称，如 "RDP - Server A"
    public string Username { get; set; }
    public string EncryptedPassword { get; set; } // Base64 存储的密文
    // 运行时临时解密用的属性，不序列化
    [JsonIgnore] public string DecryptedPasswordProxy { get; set; } 
}

```

**加密算法**:
必须使用 Windows 原生 **DPAPI (Data Protection API)**，无需用户设置主密码。

* **Encrypt**: `ProtectedData.Protect(bytes, null, DataProtectionScope.CurrentUser)`
* **Decrypt**: `ProtectedData.Unprotect(...)`

### 3.4 核心算法：InjectorService.cs

这是本功能实现的**最高风险点**，请严格实现以下逻辑流：

```csharp
public async Task ExecuteAsync(LauncherItem item)
{
    // 1. 获取凭据
    var secret = _credentialManager.GetSecret(item.Tag); // Tag 存储 SecretID
    if (secret == null) return;

    // 2. 准备阶段 (在 Pulsar 窗口还显示时)
    var targetWindow = _windowService.LastActiveWindowHandle; // 必须在 Show() 时记录好的句柄

    // 3. 关键：通知主窗口关闭，并 "阻止" 主窗口默认的焦点归还逻辑
    // (因为我们要在这里手动控制时序，防止竞态条件)
    _eventAggregator.Publish(new CloseWindowMessage { 
        ActionExecuted = true // 设为 true，告诉 MainWindow 别乱动焦点
    });

    // 4. 手动归还焦点 (Focus Boomerang)
    NativeMethods.SetForegroundWindow(targetWindow);
    
    // 5. 缓冲区等待 (Wait Buffer)
    // 必须等待 Windows 动画完成和输入队列就绪。
    // 建议值: 100ms - 200ms
    await Task.Delay(150); 

    // 6. 执行注入 (使用 SendKeys 或 InputSimulator)
    SendKeys.SendWait("^a"); // Ctrl+A 全选
    SendKeys.SendWait("{BACKSPACE}"); // 清空
    SendKeys.SendWait(secret.Username);
    SendKeys.SendWait("{TAB}");
    // 稍微停顿，防止 Tab 切换动画未完成
    await Task.Delay(50); 
    SendKeys.SendWait(secret.DecryptedPasswordProxy);
    SendKeys.SendWait("{ENTER}");
}

```

## 4. UI 交互需求 (UI/UX)

### 4.1 设置界面 (Settings Window)

* 在 `SettingsWindow` 中增加一个 Tab 或区域：**"Key Injector"**。
* **列表**: 显示已保存的 `Alias` 和 `Username`。
* **新增/编辑弹窗**: `QuickSecretsDialog.xaml`
* 输入框: Alias, Username, Password (使用 `PasswordBox`).
* 保存时自动调用 DPAPI 加密并写入 `secrets.json`。



### 4.2 轮盘配置 (Radial Menu Setup)

* 在现有的 `ProcessPickerDialog` (或新建一个 `ActionPicker`) 中，允许用户选择 "Type: PKI Secret"。
* 下拉选择已配置的 Secret Alias。
* 生成的 `LauncherItem`:
* `Type`: `GridItemType.PkiSecret`
* `Tag`: `SecretItem.Id` (关联 ID)
* `Icon`: 默认显示一个钥匙图标或用户自定义。

## 5. 验收标准 (Acceptance Criteria)

1. **MSTSC 测试**: 打开 Windows 远程桌面，点击账号框，呼出 Pulsar，选择凭据。程序应自动填入并回车登录。
2. **安全性测试**: 打开生成的 `secrets.json` 文件，密码字段必须是乱码（Base64 密文）。
3. **焦点稳定性**: 在执行注入时，Pulsar 窗口必须完全消失，且不会因为输入过快导致部分字符打在 Pulsar 窗口上。
4. **架构合规**: 代码必须位于 `Features/Pki` 目录下，且通过 `ILauncherAction` 接口调用，不得在 `CommandService` 中写死 `if (type == "PKI")` 的硬编码逻辑。
