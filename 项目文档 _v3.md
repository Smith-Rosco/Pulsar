##1. 🏗️ 项目全景与蓝图 (Project Blueprint)* **项目名称**: Pulsar (脉冲星)
* **项目愿景**: 构建一个基于现代 WPF 技术栈的系统级效率轮盘。通过严格的 MVVM 架构实现 UI 与逻辑分离，利用依赖注入管理服务生命周期，提供极致的响应速度与“果冻般”的视觉体验。

###核心架构变革 (Architecture Evolution)| 特性 | 旧版 GridCaster | **新版 Pulsar** |
| --- | --- | --- |
| **架构模式** | Monolithic (UI 逻辑耦合) | **Strict MVVM** (UI 与逻辑分离, 状态驱动) |
| **依赖管理** | 静态类 / 单例 | **DI 容器** (Microsoft.Extensions.DI) |
| **生命周期** | 隐式关闭 (OnMainWindowClose) | **显式管理** (OnExplicitShutdown + TrayIcon) |
| **交互逻辑** | 按下显示 -> 松开Q执行 | **按住维持** (Hold Ctrl -> Hover -> Release Ctrl) |
| **渲染机制** | 简单的 MouseMove 事件 | **60FPS 渲染循环** (DispatcherTimer + VisualTreeHelper) |

##3. 📜 技术契约与规范 (Technical Contracts)###3.1 命名空间管理 (Critical)* **混合框架冲突**: 项目同时引用了 WPF 和 WinForms (用于托盘/鼠标/钩子/SendKeys)。
* **解决方案**: `GlobalUsings.cs` 或文件头部显式定义：
```csharp
using WpfApplication = System.Windows.Application;
using Forms = System.Windows.Forms; 
using GridItem = Pulsar.Models.GridItem; // 区分 System.Windows.Forms.GridItem

```



###3.2 交互逻辑规范 (Interaction Logic)* **Hold-to-Maintain (按住维持)**:
* **触发**: `Ctrl + Q` -> `Show()`。
* **上下文感知**: `Show()` 时调用 `WindowService.GetForegroundWindow()` 获取当前进程名，动态加载 Profile 数据。
* **执行**: `Ctrl` 抬起 -> `ExecuteSelection()` -> `Hide()`。


* **状态重置**: 每次 `Show()` 必须调用 `ResetSelection()`，清除 ViewModel 中的 `_activeSlotIndex` 残留。

###3.3 智能命令解析 (Smart Command Parsing)* **参数支持**: `CommandService` 不能直接将 `Cmd` 传给 `Process.Start`。
* **解析策略**:
1. **显式 .exe**: 查找字符串中的 `.exe`，以此为界拆分 `FileName` 和 `Arguments`。
2. **空格探测**: 如果无 `.exe` 但文件不存在，尝试以第一个空格拆分。
3. **智能切换**: `Launcher` 类型下，优先尝试 `SwitchToProcessAsync`，失败后再 `LaunchApplicationAsync`。



###3.4 窗口互操作 (Native Interop)* **前台检测**: 使用 `GetForegroundWindow` + `GetWindowThreadProcessId` 获取当前用户正在操作的软件。
* **强制切换**: 使用 `AttachThreadInput` (可选) 或 "Alt键欺骗法" (`keybd_event`) 绕过 Windows 焦点保护，确保窗口能被强制置顶。
* **P/Invoke 别名**: 在 `WindowService` 中使用 `EntryPoint` 重命名原生方法 (如 `GetForegroundWindow_Native`)，避免与接口方法名冲突。

##5. 🛠️ 故障排查指南 (Troubleshooting)* **Q: SettingsWindow 打开后点击按钮报错 `XamlParseException`?**
* **A**: `Effect` 属性绑定了布尔值 (`DropShadowKey`)。**修复**: 移除错误的 `Effect` 绑定，或使用具体的 `DropShadowEffect` 对象。


* **Q: 无法运行带参数的命令 (如 `app.exe arg1`)?**
* **A**: `Process.Start` 将整串字符视为文件名。**修复**: 在 `CommandService` 中恢复 `ParseCommand` 逻辑，智能拆分 EXE 路径和参数。


* **Q: 切换到 Chrome 等已打开程序时无反应？**
* **A**: `WindowService` 曾被简化为桩代码。**修复**: 恢复 `FindWindowByProcess` 和 `EnumWindows` 逻辑，并实现 `ForceForegroundWindow`。


* **Q: 编译报错 "WindowService.GetForegroundWindow 具有二义性"?**
* **A**: 接口方法名与 DllImport 方法名冲突。**修复**: 给 DllImport 方法添加 `EntryPoint` 并重命名 C# 对应的私有静态方法 (如 `_Native`)。
